## What This Repository Does

This is the **Blue Dots Economy** deployment automation repo. It provisions AWS infrastructure and deploys a three-tier Kubernetes application stack using OpenTofu/Terragrunt + Helm.

## Repository Structure

```
helm/
  common-services/   # Cluster-wide: ingress-nginx, cert-manager, PostgreSQL 17, Redis 7
  signals/           # DPG app: Fastify API, Vite/React UI, notification, match-score
  aggregator/        # Portal: Node web BFF, Fastify API, BullMQ worker, Keycloak
opentofu/aws/
  template/          # Reference environment — clone to dev/staging/prod/
  modules/           # Reusable OpenTofu modules
  _common/           # Shared Terragrunt config
  <env>/             # Each environment has its own install.sh entrypoint
```

## Commands

All operations go through `install.sh` in the environment directory (`opentofu/aws/<env>/install.sh`). Functions are chainable:

```bash
# Infrastructure
bash install.sh create_tf_backend       # Create S3 remote state bucket
bash install.sh create_tf_resources     # Run terragrunt apply (VPC → EKS → IAM → storage → secrets)

# Helm
bash install.sh deploy_common_services  # Install shared platform stack first
bash install.sh deploy_signals          # Install DPG app
bash install.sh deploy_aggregator       # Install portal app
bash install.sh deploy_all_services     # Full chain in dependency order

# Validation
bash install.sh lint                    # Helm lint all three charts
bash install.sh dry_run                 # Helm --dry-run for all charts
bash install.sh preflight              # Pre-deployment checks

# Teardown
bash install.sh cleanup_all_services   # Remove Helm releases + namespaces
bash install.sh destroy_tf_resources   # Tear down AWS infrastructure
```

## Architecture

### Deployment Order (strict dependency chain)

1. **common-services** (`common-services` namespace) — must deploy first; provides ingress-nginx LoadBalancer, cert-manager, shared Postgres, and Redis used by all other charts
2. **signals** (`signals` namespace) — DPG monorepo; connects to `dpg-postgres.common-services.svc.cluster.local`
3. **aggregator** (`aggregator` namespace) — Blue Dots portal; connects to `aggregator-postgres.common-services.svc.cluster.local` and includes Keycloak with a custom OTP SPI

### Config Generation Pattern

OpenTofu generates one root-level YAML values file per Helm chart via the `output-file` module. These files are passed directly via `-f` flags — no yq splicing needed. The generated files are git-ignored.

### Environment Setup

`opentofu/aws/template/` is the canonical reference. New environments are created by copying this directory. The `global-values.yaml` in each environment holds all environment-specific inputs (region, EKS sizing, hostnames, S3 buckets, subnet CIDRs).

### Keycloak Realm Rendering

A pre-install Helm hook runs an `alpine/sed` initContainer (`realm-renderer`) that substitutes placeholders into `aggregator-realm.json` before Keycloak starts.

### Secrets

Random passwords are generated by OpenTofu (`random_id`, `random_password`) and templated into each chart's values file. The same OpenTofu variable appears in multiple rendered files so cross-namespace services share consistent credentials.

## Prerequisites

- OpenTofu 1.6.0+, Terragrunt 0.90+
- AWS CLI 2.0+, `kubectl` 1.24+, Helm 3.12+
- GHCR PAT with `read:packages` scope (for pulling images from `ghcr.io/blue-dots-economy/`)
- DNS control for the public hostnames defined in `global-values.yaml`

## Image Registry

All application images are at `ghcr.io/blue-dots-economy/`. Signals images: `signals-dpg/{api,ui}`. Aggregator images: `aggregator/{web,api,worker,keycloak}`.
